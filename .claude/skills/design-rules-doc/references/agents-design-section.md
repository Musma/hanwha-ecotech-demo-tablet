## Design Rules

- UI를 새로 구현하거나 기존 UI를 수정하기 전에는 반드시 `DESIGN.md`를 먼저 읽는다.
- `DESIGN.md`는 디자인 가이드와 프로젝트 구조를 연결한 UI 구현 기준이다.
- 색상, 타이포그래피, spacing, radius, shadow, 아이콘, 레이아웃, 반응형 동작, 컴포넌트 사용 방식은 `DESIGN.md`를 우선한다.
- 새 UI는 기존 토큰과 공용 컴포넌트를 먼저 사용하고, 임의의 시각 값을 추가하지 않는다.
- `DESIGN.md`, `ARCHITECTURE.md`, `FOLDER_STRUCTURE.md`, 기존 구현이 충돌하면 임의 판단하지 말고 사용자에게 보고한다.
